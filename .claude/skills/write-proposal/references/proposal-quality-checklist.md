# Proposal Quality Checklist

Use this checklist during Phase 4 (Structural and Conceptual Validation) before language polishing.

## Golden Thread

- [ ] A single overarching research aim is stated
- [ ] Each objective traces back to the aim
- [ ] Each research question traces back to an objective
- [ ] Each methodology component answers a specific research question
- [ ] The expected contribution addresses the stated gap

## Contribution Integrity

- [ ] The proposal has exactly one primary contribution; all other components are labeled as supporting, validation, or future work
- [ ] Every novel construct (index, metric, framework, equivalence relation) has been classified as: theoretical construct / methodological device / design artifact / planning tool
- [ ] No construct derived primarily from simulation sweeps is labeled a "theoretical contribution" or "formal characterization" without a stated theorem, proof obligation, or invariance property
- [ ] Claim strength language matches the evidence level (strong / moderate / cautious)
- [ ] Phrases such as "formal theory", "validated framework", "principled basis", "theoretical contribution" are only used where the evidence supports a strong claim

## Scope

- [ ] The proposal contains no more than one primary novel technical component
- [ ] If more than two novel components appear, one has been designated primary and the rest moved to supporting components or future work
- [ ] The minimum viable PhD contribution is explicitly identifiable from the proposal

## Methodology Integrity

- [ ] Every research question is answered by a named, specific method
- [ ] If data sources span multiple abstraction levels, a bridge table has been completed and its logic is reflected in the methodology and limitations sections
- [ ] All high-risk and unverifiable assumptions are acknowledged in the limitations section
- [ ] Cross-scale or cross-layer inferences are explicitly justified

## Substitutability and Equivalence

- [ ] Any substitutability or equivalence claim is decomposed into specific value dimensions
- [ ] Non-substitutable dimensions are stated
- [ ] Boundary conditions and saturation points are stated
- [ ] The claim is labeled conditional or general as appropriate

## Candidate and Supervisor Fit

- [ ] The proposal methods are credible given the applicant's CV
- [ ] The proposal builds on (not merely cites) the professor's specific methods or open questions
- [ ] The professor would see a clear reason to supervise this project

## Citation Function

- [ ] Context-setting references are present in the introduction
- [ ] Gap-establishing references are present in the problem statement and literature review
- [ ] Method-justifying references are present in the methodology
- [ ] Benchmark or dataset references are present where applicable
- [ ] Limitation-aware references are present in the limitations section
- [ ] No citation is included solely to increase bibliography size

## Research Methodology Detail

- [ ] Research design is clearly stated and justified
- [ ] Sampling method is described (or marked N/A for computational work)
- [ ] Data collection methods and instruments are detailed
- [ ] Data analysis techniques are specified and linked to each RQ
- [ ] Methodological limitations are acknowledged

## Ethical Considerations

- [ ] Ethical issues are identified and addressed (or a brief statement that none apply is provided)

## Timeline and Budget

- [ ] Realistic timeline with major milestones is present
- [ ] Budget is included if required (or marked N/A)

## Mechanical Cleanliness

- [ ] No encoding artifacts (corrupted punctuation, broken apostrophes, broken dashes)
- [ ] No malformed mathematical notation
- [ ] No Markdown table corruption
- [ ] No citation-reference mismatches (every in-text citation has a reference list entry and vice versa)
- [ ] No placeholder remnants (e.g., [INSERT], TBD, [Author, Year])
- [ ] No duplicate headings
- [ ] Terminology is consistent throughout (core concepts referred to by the same name)
- [ ] No broken file names or output paths
